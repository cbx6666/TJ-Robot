# 实验原理详解：建图、导航、识人与「去人」

本文说明 **TJ-Robot** 仓库中**仿真栈**与 **ROS 2** 各模块如何协同：激光建图、YOLO 识人、人物方位与激光对齐、以及两种减少人进地图的实现路径。可与 **[实验报告_AI语义识别剔除动态障碍物建图对照.md](./实验报告_AI语义识别剔除动态障碍物建图对照.md)** 对照阅读。

---

## 1. 总体架构

### 1.1 分层视图

| 层次                   | 作用                   | 本仓库主要组件                                                                    |
| ---------------------- | ---------------------- | --------------------------------------------------------------------------------- |
| **仿真**         | 物理与传感器           | Gazebo、`test1.world`、TurtleBot3 Burger（assist 下带仿真 RGB）、动态障碍物插件 |
| **状态与 TF**    | 关节、坐标系           | `robot_state_publisher`、`/tf`、`/tf_static`                                |
| **建图**         | 2D 占据栅格            | slam_toolbox，订阅 `/scan` 或（可选）`/scan_filtered`                         |
| **语义感知**     | 谁在画面里、在哪个方向 | YOLO-Seg（`human_yolo_seg`），输出检测框/分割与 人物方位角区间                  |
| **人–激光融合** | 把「人」对应到激光角域 | 角区间话题 + 时间同步 +（strip）激光点投 map                                      |
| **去人**         | 减轻动态人导致的错占据 | 事后 strip 地图 或 实时滤扫                                                       |
| **导航**         | 覆盖行走、点到点、Nav2 | `coverage_patrol` / `point_to_point` / `coverage_patrol_nav2`               |

一键入口：**`scripts/tb3_stack.sh`** 负责按环境变量拉起 Gazebo、机器人、SLAM、RViz、可选 YOLO 人物链。

### 1.2 话题与坐标系（核心）

- **`/scan`**：`sensor_msgs/LaserScan`，`frame_id` 一般为 **`base_scan`**。每一束激光用 **`angle_min + i × angle_increment`** 表示方向，与 **`atan2(y, x)`** 在激光系下的方位一致（**x 前、y 左**，左侧对应正角）。
- **`/camera/image_raw`**、**`/camera/camera_info`**：RGB 与内参；光学系一般为 **`camera_rgb_optical_frame`**，经 TF 与车体、激光关联。
- **`/human_yolo/person_azimuth_ranges`**：`sensor_msgs/JointState`，**`position = [lo₀, hi₀, lo₁, hi₁, …]`**（弧度），表示在 **与 `/scan` 相同约定** 下的**水平角区间**：人大致占据哪些激光方向。
- **`/map`**：slam_toolbox 发布的占据栅格，**`frame_id = map`**。

## 2. 建图原理

### 2.1 SLAM 输入

本实验采用 **2D 激光 SLAM（slam_toolbox）**，不依赖深度相机建图。前端输入为 **激光扫描**：

- **默认 / 推荐对照实验**：**`mapper_params_online_async_full_scan.yaml`**，订阅 **`/scan`**。与「仅激光基线」一致，便于公平对比。
- **filtered 模式**：参数文件切换为 **`mapper_params_online_async_scan_filtered.yaml`**，订阅 **`/scan_filtered`**（由下文「实时滤人」节点生成）。

### 2.2 动态人对地图的影响

激光无法区分墙与人。人移动时，同一栅格会反复被标为占据/空闲，表现为 **鬼影、拖尾、错占据**。实验目的即验证：在**语义知道此处曾是人**的前提下，能否在**后处理或前端**减轻该现象。

### 2.3 `tb3_stack` 与模式

- **`TB3_STACK_MODE=laser`**：经典 Burger 激光仿真（无 assist 时是否注入 RGB 取决于是否开 YOLO，见根目录 README）。
- **`TB3_STACK_MODE=assist`**：Burger 上挂 **仿真 RGB**（与 Waffle 前相机位姿对齐思路），供 YOLO 使用。
- **`TB3_PERSON_SLAM_MODE`**：
  - **`mark_then_strip`（默认）**：SLAM **始终吃 `/scan`**；人物信息由 **`person_strip_recorder`** 并行记录，**不改动** SLAM 输入。
  - **`filtered`**：SLAM **吃 `/scan_filtered`**，建图过程中即弱化人向激光读数。

## 3. 识别人：YOLO 与人物方位角

### 3.1 检测与分割

节点 **`yolo_person_seg_node`**（包 **`human_yolo_seg`**）订阅 RGB，用 **Ultralytics YOLO-Seg** 在图像中检出 **person**（默认 COCO 类 0）。输出包括：

- 带框/分割的 **`/human_yolo/annotated_image`**（可视化）；
- 内部 **`boxes`**、可选 **`masks`**，用于几何计算。

### 3.2 从像素到「激光角区间」

需要将「人在图像中的位置」变为 **`base_scan` 下的方位角区间 [lo, hi]**，才能与 **`/scan`** 的每一束角对齐。本仓库提供两种模式（由参数 **`person_azimuth_mode`** / 环境变量 **`TB3_PERSON_AZIMUTH_MODE`** 选择）：

| 模式                                 | 原理                                                                                                                                                                                                 | 特点                                                                                                |
| ------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------- |
| **`linear_fov`（当前默认）** | 取框（或 mask 底边）的**左右像素 u**，按 **`(u − cx) / 图像宽度 × 水平视场角`** 映射为角度，并取负号与 **激光左正右负** 约定一致；可加 **`camera_yaw`** 微调安装误差。 | **不算 TF、不算地面射线**；快、依赖 CameraInfo 的 fx/视场或默认约 135°；适合仿真宽视场相机。 |
| **`tf_geometry`**            | 用内参将像素射线投到**`base_footprint` 上 z=0 地面**，再 TF 到 **`base_scan`**，对 **`atan2(y, x)`** 得角。                                                                        | 几何上更完整；依赖**TF 与标定**，延迟与同步要求更高。                                         |

输出封装为 **`JointState.position`** 的成对 **`[lo, hi]`**（每人一段区间，多人多段），**`header.stamp` 使用当前 RGB 时间戳**，供下游与激光 **时间对齐**。

### 3.3 时间同步的重要性

`person_strip_recorder`、`scan_person_filter`、`scan_map_colored_cloud` 等会检查 **|t_scan − t_image|** 是否小于阈值（如 **0.5 s**）。若不对齐，会出现「激光转过去了，角域还是上一帧人」的错位，掩膜或 strip 区域会偏。

## 4. 去除人的两种实现路径

二者目标一致：**减少人对最终地图或当前扫描的负面影响**；实现位置不同。

### 4.1 路径 A：`mark_then_strip` + 事后清图

**原理**：建图阶段 **不修改 SLAM 的 `/scan`**，与纯激光基线**同一 SLAM 前端**；并行做「人向记录」。

1. **`person_strip_recorder_node`** 订阅 **`/scan`** 与 **`/human_yolo/person_azimuth_ranges`**。
2. 当角域有效且与当前 scan 时间对齐时，对 **落在这些角区间内的、有效的激光点**，用 TF 投到 **`map`**，在栅格上 **去重累积** 成人可能出现的区域代表点。
3. 结果写入 **`~/.ros/tj_person_strip_regions.yaml`**（圆心+半径列表），并可选发布 **`/human_yolo/person_laser_map_cloud`**（黄点云，便于 RViz 对照）。
4. **建图结束、保存 `map.yaml` + `.pgm` 后**，运行 **`strip_saved_map_person_free`**：按 YAML 中圆将占据栅格 **刷为空闲**，得到**语义后处理后的地图**。

**优点**：对照实验公平（SLAM 输入与基线一致）；人的位置由 **角域 × 激光测距**共同决定，比单用像素更贴实际遮挡。
**注意**：若保存地图瞬间人仍站在某格，该格可能仍被占据；strip 依赖累积区域与检测连续性。

### 4.2 路径 B：`filtered` + 实时滤扫

**原理**：**`scan_person_filter_node`** 复制 **`/scan`**，把 **角区间内的 `ranges[i]`** 置为 **`inf` / `nan` / `range_max`**（可配），发布 **`/scan_filtered`**；SLAM 若订阅该话题，则 **建图过程中** 就少摄入人向回波。

**优点**：实时、地图前端即弱化人。
**缺点**：与仅激光建图**输入不同**，严格对照实验需单独说明；近距离若滤过头，可能影响定位（故巡航仍推荐用原始 **`/scan`** 避障）。

## 5. 导航原理

### 5.1 `coverage_patrol`（非 Nav2）

- 按 **预定义航点**（房间边、中线、可选人物区域）做 **覆盖式巡逻**。
- 订阅 **`/odom`**，发布 **`/cmd_vel`**；前方障碍用 **轻量反应式逻辑**（含脱困航点）。
- **激光避障一律用原始 `/scan`**：避免把人 **滤成透明** 导致贴人行驶，与 **`mark_then_strip`** 设计一致。

### 5.2 `coverage_patrol_nav2` + `nav2_coverage_patrol.launch.py`

- **航点生成方式**与上面类似，但每个目标交给 **Nav2 `NavigateToPose`**，由 **局部规划器 + 代价地图** 跟踪。
- 代价地图仍推荐用 **原始 `/scan`** 做动态障碍，与文档中「小空间 + 人形障碍」场景一致。

### 5.3 `point_to_point`

- 最小 **`/cmd_vel`** 闭环，用于验证底盘与坐标，不承担覆盖或语义。

**小结**：**建图**可用 strip 事后清图或 filtered 实时滤扫；**导航避障**侧默认仍强调 **真实 `/scan`**，与「去人主要针对地图可信度」分工明确。

## 6. 实验对照在原理上的含义

| 对照组（仅激光）   | 不启 YOLO 链，`TB3_ASSIST_SCAN_FILTER=0`；SLAM 仅 `/scan`；**无** strip/filter。                                                                          |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 实验组（YOLO+RGB） | 启 YOLO，得到角域；**mark_then_strip** 下 SLAM 仍 `/scan`，靠 **strip** 利用语义清图；或 **filtered** 下 SLAM 用 **`/scan_filtered`**。 |

评价指标侧重：**动态人活动区内占据栅格差异、鬼影、轨迹稳定性**等（见实验报告表格）。
